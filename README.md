# Financial Instrument Golden Copy System

## Overview
In financial institutions, instrument data often comes from multiple vendors. Even though the data refers to the same instruments, the values can sometimes be different across vendors. For example, the price of an instrument or its currency might vary slightly depending on the source. These inconsistencies can create problems for analysts and systems that depend on accurate data.
The purpose of this project was to simulate a **Golden Copy system**, which combines data from different vendors and produces a single reliable version of the dataset. Along with the data pipeline, a **Power BI dashboard** was created to monitor conflicts and data quality.

## Problem Statement
Financial data teams often deal with multiple data vendors providing similar information about financial instruments. Since each vendor may collect or update data differently, conflicts between datasets are common.
Some of the issues observed in the datasets include:
- Different prices reported by different vendors  
- Missing identifiers such as ISIN or ticker  
- Duplicate instrument records  
- Slight variations in currency or other attributes  
Without a proper reconciliation process, it becomes difficult to know which data should be trusted. The goal of this project was to build a system that identifies these conflicts and produces a **single consistent dataset** that can be used for analysis.

## About the Dataset
The dataset used in this project represents **financial instrument reference data collected from three different vendors**. Each vendor file contains information about instruments such as identifiers, prices, and currency.
Key fields included in the dataset are:
- Instrument ID  
- ISIN  
- Ticker  
- Price  
- Currency  
- Vendor source  
The datasets intentionally contain small differences between vendors so that conflict detection and reconciliation could be demonstrated.

## Approach
The project was developed as a simple data pipeline with a few clear steps:
1. Vendor datasets were first loaded into Python.
2. The data was standardized so that column names and formats matched across files.
3. Records from different vendors were compared to detect differences in values.
4. Conflicts and missing values were flagged as exceptions.
5. A final **Golden Copy dataset** was generated by applying simple reconciliation rules.
This process helped combine multiple datasets into one reliable version.

## How the Problem Was Addressed
To deal with differences between vendor data, the project focuses on identifying conflicts early in the pipeline. By comparing key attributes across vendors, the system highlights records where values do not match.
Instead of ignoring these issues, the pipeline logs them as conflicts or exceptions. This makes it easier to review problematic records and ensure the final dataset remains reliable. The Golden Copy dataset produced at the end represents the most consistent version of the data.

- Generated a consolidated **Golden Copy dataset**  
- Created a Power BI dashboard to monitor data quality and vendor differences
